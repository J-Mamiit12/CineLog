📋 Project Case Study

About CineLog

A comprehensive JavaScript-powered movie watchlist manager built for ITEL 203 — Web Systems and Technologies.

🎯 Purpose of the System

CineLog is a dynamic, client-side web application that allows users to manage their personal movie watchlist. Users can add movies with detailed metadata, track their viewing status, rate films, and filter or search through their collection — all without a backend server. Data is stored locally using the browser's localStorage API, making it persist across page reloads.

The project demonstrates proficiency in DOM manipulation, event-driven programming, form validation, data visualization (Chart.js), and modern front-end design principles as required by the course syllabus.

How the System Works

01

Data Entry & Validation

Users input movie details via a form. JavaScript listens to events and strictly validates inputs in real-time before accepting the data.

02

Local Storage API

The validated data is serialized into JSON format and saved securely in the browser's localStorage, acting as our client-side database.

03

Dynamic Rendering

The system reads the stored data and dynamically constructs HTML tables, stats, and recommendations via DOM manipulation.

System Features

  • Add movies with complete metadata
  • Interactive 5-star rating system
  • Multiple Watch Status tracking
  • Real-time form validation & alerts
  • Dynamic DOM table rendering
  • Duplicate entry detection
  • Live multi-keyword search
  • Category filtering by status
  • Interactive Chart.js analytics
  • Algorithmic recommendations
  • Animated toast notifications
  • Persistent localStorage data
